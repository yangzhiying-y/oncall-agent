### TEST 3: RAG 知识问答（重排验证）

```bash
curl -X POST http://localhost:9900/api/chat \
  -H "Content-Type: application/json" \
  -d '{"Id":"test-1","Question":"AIOps 有哪些核心能力？请基于知识库回答。"}'
```

**问题**：AIOps 有哪些核心能力？

**回答**：
> AIOps 的核心能力包括：
> 1. **故障预测** — 通过机器学习算法分析历史数据预测潜在故障
> 2. **根因分析** — 自动关联多维度数据快速定位问题根因
> 3. **自动化修复** — 基于预设策略自动执行修复操作
> 4. **智能告警** — 减少告警噪音提高告警准确率

**重排链路验证**：
- 粗筛：向量检索 Top-10 候选文档
- 精排：qwen3-rerank 模型重排，返回 Top-3
- LLM 基于精选文档生成答案，内容与上传文档一致

## 如何自行测试

### 前置条件

1. **Docker Desktop 已启动**
2. **Milvus 容器运行中**：`docker ps | grep milvus`
3. **FastAPI 服务已启动**：`start-windows.bat` 或手动启动
4. **API Key 有效**：`.env` 中的 `DASHSCOPE_API_KEY` 有 qwen3-rerank 权限

### 速测脚本

```bash
# 1. 冒烟测试
curl http://localhost:9900/health

# 2. 上传文档
curl -X POST http://localhost:9900/api/upload -F "file=@your_doc.md"

# 3. 知识问答（触发重排）
curl -X POST http://localhost:9900/api/chat \
  -H "Content-Type: application/json" \
  -d '{"Id":"test","Question":"你的问题"}'

# 4. AIOps 诊断
curl -X POST http://localhost:9900/api/aiops \
  -H "Content-Type: application/json" \
  -d '{"session_id":"test"}' --no-buffer
```

### 查看日志

```powershell
# Windows - Loguru 日志按天轮转
Get-Content logs\app_$(Get-Date -Format "yyyy-MM-dd").log -Tail 50

# 关注重排日志
Get-Content logs\app_$(Get-Date -Format "yyyy-MM-dd").log | Select-String "重排"
```

### 重排效果验证

观察日志中的关键信息：
```
粗筛阶段检索到 X 个候选文档    ← 第一阶段
重排完成 - 返回 X 个文档       ← 第二阶段 (qwen3-rerank)
```

文档格式化输出中会包含 `相关度: 0.XXXX` 字段。
